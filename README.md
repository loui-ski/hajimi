# 【保姆级教程】VPN 速度慢？只能用 TCP？两招“物理外挂”拯救你的网速

如果你身处网络审查极其严格的环境（如 GFW 下），被迫使用 **OpenVPN (TCP) + Scramble (混淆)** 这种组合，你一定经历过绝望：明明宽带很快，但连上 VPN 后速度只有几百 KB，视频卡顿，甚至频繁断流。

这不是你的宽带问题，也不是 VPN 垃圾，而是因为**数据包太大，卡在路上了**。

本文将分享一套经过实测验证的\*\*“底层抗干扰”优化方案\*\*，不需要你懂复杂的网络代码，只需要两步操作，就能大幅提升稳定性。

-----

## 核心原理：为什么你的 VPN 这么慢？

在受干扰的网络环境下，我们被迫使用 **TCP 协议**。TCP 就像运货卡车，它非常讲究“顺序”和“确认”。

1.  **TCP 崩溃效应 (TCP Meltdown)：** 当网络丢包时，TCP 会傻傻地等待重传，导致后面的数据全部堵塞。
2.  **分片灾难：** 默认的网络包大小 (MTU) 是 1500。加上 VPN 的加密壳子，数据包变得臃肿。一旦超过路宽，就会被路由器强制“切片”。切片+丢包 = 速度雪崩。

**解决思路：** 我们不能把路修宽，但我们可以**把车改小**。我们将强制电脑只发送“迷你数据包”，让它像摩托车一样在拥堵的网络车流中灵活穿梭。

-----

## 第一步：修改物理网卡 MTU（核心大招）

很多教程让你测最佳 MTU（通常是 1472），**请忽略它们！** 那是给直连用户用的。对于抗干扰 VPN 用户，我们需要极端保守的数值：**1200**。

### 操作步骤：

1.  **以管理员身份打开 CMD**

      * 按 `Win` 键，输入 `cmd`，右键点击“命令提示符”，选择 **“以管理员身份运行”**。

2.  **找到你的网卡名称**

      * 输入命令：`netsh interface ipv4 show subinterfaces`<img width="518" height="175" alt="{4DA45DA1-5A64-4FB6-8979-A80CEE1B71DE}" src="https://github.com/user-attachments/assets/080dab8c-0065-4de2-927a-29da39054570" />

      * 查看 **“接口”** 这一列。
      * 如果你用 Wi-Fi，通常叫 **`WLAN`**。
      * 如果你插网线，通常叫 **`以太网`** 或 **`Ethernet`**。
      * *注意：找到那个状态为“已连接”且流量巨大的接口。*

3.  **强制降级 MTU 为 1200**

      * 输入以下命令（注意将 `"WLAN"` 替换为你真实的接口名称）：
        ```cmd
        netsh interface ipv4 set subinterface "WLAN" mtu=1200 store=persistent
        ```
      * 提示“确定”即为成功。

4.  **重启 VPN**

      * 断开并重新连接你的 VPN，你会发现连接速度变快了，且不容易断线。

> **Q：这会影响我日常不挂 VPN 上网吗？**
> A：几乎没影响。虽然理论极限下载速度会降低 3% 左右，但在浏览网页和打游戏时，小包反而能降低延迟（Bufferbloat），手感甚至更好。

-----

## 第二步：关闭浏览器 QUIC 协议（锦上添花）

Chrome 和 Edge 浏览器默认使用 **QUIC 协议**（基于 UDP）来加载 Google/YouTube 视频。
当你在 VPN 的 TCP 隧道里跑 UDP 流量时，会发生严重的**协议打架**，导致视频缓冲条一卡一卡。我们要关掉它，强迫浏览器走 TCP。

### 操作步骤：

1.  打开浏览器（Chrome 或 Edge）。
2.  在地址栏输入以下地址并回车：
      * **Chrome 用户：** `chrome://flags/#enable-quic`<img width="552" height="64" alt="{6094A548-630B-4518-A41F-CE4F368CEEFF}" src="https://github.com/user-attachments/assets/8e0608db-8d04-4e7f-8fe3-a1b4f0d67114" />

      * **Edge 用户：** `edge://flags/#enable-quic`
3.  找到高亮的 **Experimental QUIC protocol**。
4.  将右侧的选项从 `Default` 改为 **`Disabled`** (禁用)。
5.  点击右下角的 **Relaunch** 重启浏览器。

-----

## 总结

通过这两步操作：

1.  **物理层 (MTU 1200)：** 保证数据包足够小，拒绝分片，大幅减少丢包重传。
2.  **应用层 (Disable QUIC)：** 消除“UDP over TCP”的内耗，让视频流更顺畅。

这套方案是专门针对**恶劣网络环境 + 强制 TCP 模式**的特效药。如果你也在为 VPN 速度发愁，不妨试一试！
